# 2026-04-05 OpenClaw 接入与外部速记能力

## 本次目标

让映记支持接入 OpenClaw 小龙虾，使用户可以在手机端通过自然语言把内容写入映记。

## 核心决策

这次没有直接复用用户网页登录态，也没有做一个写死的 OpenClaw 特判逻辑。  
而是实现了一层通用的 **外部速记接入能力**：

- 用户生成专属接入令牌
- 后端只保存令牌哈希
- 外部代理携带令牌即可写入当前用户的日记

这样既更安全，也为后续接入快捷指令、机器人、自动化工作流预留了统一入口。

## 设计考虑

### 为什么不用主登录 token

因为外部代理通常运行在用户自己或第三方环境里，若直接使用网页登录态：

- 泄露风险更高
- 生命周期不好控制
- 权限边界不清晰

专用接入令牌更适合这种“只允许写入日记”的场景。

### 为什么要支持 append_today

手机端随手记录通常是碎片化的，一天里可能发很多次。  
如果每次都新建一篇，会把日记切得太碎。  
所以默认更适合：

- 有今天的日记就追加
- 没有则创建
