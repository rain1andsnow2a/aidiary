# Pip代理配置指南

## 问题说明

在Windows环境下使用pip安装Python包时，如果本地开启了代理（如Clash、V2Ray等），需要配置pip使用代理，否则会出现连接错误。

## 快速解决方案

### 方法1: 临时使用代理（推荐）

**安装单个包时**:
```bash
pip install 包名 --proxy=http://127.0.0.1:10090 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**安装requirements.txt时**:
```bash
export HTTP_PROXY=http://127.0.0.1:10090
export HTTPS_PROXY=http://127.0.0.1:10090
pip install -r requirements.txt --proxy=http://127.0.0.1:10090 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 方法2: 配置pip永久使用代理

**创建或编辑pip配置文件**:

Windows配置文件位置:
```
C:\Users\你的用户名\pip\pip.ini
```

**pip.ini 内容**:
```ini
[global]
proxy = http://127.0.0.1:10090
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
trusted-host = pypi.tuna.tsinghua.edu.cn
```

**创建步骤**:
1. 打开文件夹: `C:\Users\你的用户名\pip\`（如果不存在则创建）
2. 创建文件: `pip.ini`
3. 复制上面的内容到文件中
4. 保存文件

**之后就可以直接使用**:
```bash
pip install 包名
```

### 方法3: 使用环境变量（推荐用于项目）

在项目根目录创建 `.env` 文件或添加到系统环境变量：

**Windows临时设置（当前会话有效）**:
```bash
set HTTP_PROXY=http://127.0.0.1:10090
set HTTPS_PROXY=http://127.0.0.1:10090
pip install 包名
```

**Windows永久设置**:
1. 右键"此电脑" → "属性"
2. "高级系统设置" → "环境变量"
3. 在"用户变量"中新建:
   - 变量名: `HTTP_PROXY`
   - 变量值: `http://127.0.0.1:10090`
   - 变量名: `HTTPS_PROXY`
   - 变量值: `http://127.0.0.1:10090`

## 故障排查

### 问题1: Cannot connect to proxy

**原因**: 代理未启动或端口错误

**解决**:
1. 检查代理软件是否运行
2. 确认代理端口是否为10090
3. 尝试在浏览器中访问代理管理页面

### 问题2: ProxyError

**原因**: pip尝试使用代理但连接失败

**解决**:
```bash
# 临时禁用代理
set HTTP_PROXY=
set HTTPS_PROXY=
pip install 包名 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题3: SSL证书错误

**解决**:
```bash
pip install 包名 --trusted-host pypi.tuna.tsinghua.edu.cn
```

## 代理软件配置建议

### Clash
- 端口设置: `允许局域网连接`
- HTTP端口: `10090`
- SOCKS5端口: `10091`

### V2Ray
- 代理协议: `HTTP`
- 端口: `10090`
- 监听地址: `127.0.0.1`

## Windows PowerShell 使用代理

**PowerShell设置代理**:
```powershell
$env:HTTP_PROXY="http://127.0.0.1:10090"
$env:HTTPS_PROXY="http://127.0.0.1:10090"
pip install 包名
```

## Git 配置代理（可选）

如果Git也需要使用代理:
```bash
git config --global http.proxy http://127.0.0.1:10090
git config --global https.proxy http://127.0.0.1:10090
```

取消代理:
```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```
