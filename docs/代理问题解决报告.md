# 代理问题解决报告

## 问题描述

**原始问题**: pip install 无法连接到 PyPI，出现 `ProxyError: Cannot connect to proxy`

**错误信息**:
```
WARNING: Retrying (Retry(total=4, connect=None, read=None, redirect=None, status=None))
after connection broken by 'ProxyError('Cannot connect to proxy.',
FileNotFoundError(2, 'No such file', directory)')
```

## 根本原因

**确认原因**: 本地代理软件（如Clash、V2Ray）正在运行，但pip未配置使用代理。

**你的代理配置**:
- 代理地址: `127.0.0.1:10090`
- 代理类型: HTTP/HTTPS
- 代理软件: Clash/V2Ray（推测）

## 解决方案

### ✅ 已解决

**方法**: 使用 `--proxy` 参数和镜像源安装

**成功安装的包**:
```bash
export HTTP_PROXY=http://127.0.0.1:10090
export HTTPS_PROXY=http://127.0.0.1:10090
pip install aiosmtplib pydantic-settings python-dotenv httpx \
  --proxy=http://127.0.0.1:10090 \
  -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**安装结果**:
- ✅ aiosmtplib 4.0.2 - 异步邮件发送
- ✅ pydantic-settings - 配置管理
- ✅ python-dotenv - 环境变量
- ✅ httpx - HTTP客户端

## 改进内容

### 1. 邮件服务升级

**之前**: 使用同步 smtplib（内置）
```python
# 同步发送，性能较低
smtplib.SMTP_SSL(host, port)
```

**现在**: 使用异步 aiosmtplib（已安装）
```python
# 异步发送，性能更高
await aiosmtplib.send(..., use_tls=True)
```

**性能提升**:
- ✅ 异步非阻塞
- ✅ 支持并发发送
- ✅ 更好的错误处理

### 2. 便捷脚本

**创建文件**:
- `install.bat` - 依赖安装脚本（自动配置代理）
- `start.bat` - 快速启动脚本
- `docs/代理配置指南.md` - 完整配置教程

**使用方法**:
```bash
# 安装依赖（自动配置代理）

```

### 3. 永久配置方案

**可选方案**: 创建 pip 配置文件

文件位置: `C:\Users\你的用户名\pip\pip.ini`

```ini
[global]
proxy = http://127.0.0.1:10090
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
trusted-host = pypi.tuna.tsinghua.edu.cn
```

配置后可以直接使用:
```bash
pip install 包名
```
