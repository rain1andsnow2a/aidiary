# AI 综合分析

> 潜入你的冰山之下——让 AI 帮你回顾一段时间的日记，用萨提亚冰山模型给出深度洞察。

---

## 模块定位

AI 综合分析是一个**按需触发**的功能。用户主动点击「开始分析」后，后端通过 **RAG 证据收集 + 多智能体萨提亚冰山分析**，从时间窗口内的日记中提炼出行为模式、情绪趋势、深层信念和内心渴望，最终以**冰山卡片**的形式呈现。

与成长中心的区别：成长中心是**被动展示**（打开就能看），综合分析是**主动请求**（需要等待生成）。

路由：`/analysis`（需登录）

---

## 两条分析路径

| | 综合分析 | 当日分析 |
|---|---------|---------|
| **触发** | 用户在 `/analysis` 页面主动发起 | 每日 0 点定时任务自动执行 |
| **范围** | 时间窗口内的多篇日记（默认 90 天） | 仅当天的日记 |
| **深度** | 多智能体萨提亚冰山（5 层） | 轻量：提取情绪标签 + 事件 + 简短回应 |
| **呈现** | 冰山卡片 + 致你的一封信 | 数据写入成长中心，不单独展示 |

---

## 使用流程

1. 用户选择分析的**时间窗口**（默认 90 天）
2. 点击「开始分析」
3. 等待后端多轮大模型调用（通常 20~50 秒）
4. 页面以冰山卡片形式展示分析报告

---

## 后端分析流程

前端调用 `POST /api/v1/ai/comprehensive-analysis`，后端依次执行三个阶段：

### Phase 1：RAG 证据收集

```
1. 从数据库读取时间窗口内的所有日记（最多 120 篇）
2. 用 RAG 服务将日记切片 → 生成摘要块 + 原文块
3. 按 5 个维度分别检索相关片段：
   ├── 情绪趋势（焦虑、平静、开心、低落...）
   ├── 连续事件（重复出现的困扰/问题）
   ├── 关键转折（决定、变化、契机）
   ├── 成长线索（进步、反思、改变）
   └── 关系主题（家人、同事、朋友...）
4. 去重后保留最多 18 条证据片段
```

#### RAG 检索细节

使用 `rag_service.py` 实现的本地轻量 RAG，不依赖外部向量数据库：

- **切片**：日记按句子边界分段（最大 260 字，40 字重叠）
- **检索**：BM25 关键词匹配 + 多维加权排序
- **排序公式**：`55% 文本相关性 + 14% 时间新近度 + 9% 重要性 + 8% 情绪强度 + 8% 主题重复度 + 4% 人物命中 + 2% 来源加成`
- **去重**：Jaccard 相似度 ≥ 72% 的片段去重，每篇日记最多取 2 条

### Phase 2：多智能体萨提亚冰山分析

由编排器 `AgentOrchestrator` 协调多个智能体，基于 Phase 1 的证据片段，逐层分析：

```
Step A → 行为模式识别智能体
         输入: RAG 证据片段
         输出: 跨日记的重复行为聚类（如"总是深夜写日记""回避社交"）

Step B → 情绪层分析智能体
         输入: 证据片段 + Step A 结果
         输出: 情绪趋势描述 + 转折点识别 + 情绪色块数据

Step C → 认知层分析智能体
         输入: 证据片段 + Step A/B 结果
         输出: 反复出现的思维模式 / 自动化想法（如"觉得自己不够好"）

Step D → 信念层分析智能体
         输入: 所有前序结果
         输出: 深层核心信念 + 自我叙事主线

Step E → 渴望层 + 疗愈信生成智能体
         输入: 完整冰山分析结果
         输出: 内心需求描述 + 「致你的一封信」
```

每个智能体独立调用大模型，结果逐步传递给下一个。

### Phase 3：结构化报告输出

将五层分析结果 + 疗愈信 + 证据引用组装为前端可渲染的 JSON 结构。

### 视觉设计要点

- **渐变色**：卡片背景从上到下：浅蓝 → 蓝 → 靛蓝 → 深紫 → 暗金，模拟从水面潜入深海
- **展开证据**：每层卡片可展开，显示 AI 引用的具体日记片段（带日期和出处）
- **情绪色块条**：情绪层卡片中嵌入一个横向色块条，用暖色/冷色表达情绪走势
- **滚动体验**：整页向下滚动 = 向内心深处下潜，每张卡片进入视口时有淡入动画
- **致你的一封信**：最底部的疗愈性回复，用手写风格字体 + 信纸纹理背景，像一封真正的来信
- **分享**：支持将冰山卡片导出为长图分享

## 当日分析（轻量路径）

每日 0 点定时任务自动处理当天和昨天的日记，不走完整萨提亚：

```
1. 提取情绪标签 + 重要性评分
2. 提取时间轴事件（结构化）
3. 生成简短温暖回应
```

结果写入数据库，供成长中心展示。不单独生成页面。

---

## 记忆与数据积累

| 使用场景 | 是否使用历史日记 | 记忆方式 |
|----------|----------------|---------|
| 综合分析 `/analysis` | ✅ 读取时间窗口内的全部日记 | 本地 RAG（BM25 检索）→ 多智能体分析 |
| 映记精灵对话 | ✅ 检索相关日记片段 | Qdrant 向量检索（如已配置）或本地 RAG 兜底 |
| 当日分析（定时任务） | ❌ 只处理当天日记 | 轻量提取，不走 RAG |

> Qdrant 需要在 `.env` 中配置 `QDRANT_URL` 和 `QDRANT_API_KEY` 才会启用，未配置时所有检索回退到本地 BM25 RAG。

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 时间窗口 | 90 天 | 分析最近多少天的日记 |
| 最大日记数 | 120 篇 | 避免 token 超限，超过时取最近的 |

---

## 交互细节

- 点击「开始分析」后显示冰山下潜加载动画，无法重复触发
- 分析期间逐步展示已完成的层级（Step A 完成 → 行为层卡片先出现）
- 分析失败时显示错误提示，可重试
- 报告生成后停留在页面上，刷新页面后需重新分析（不缓存）
- 左上角返回按钮 → 回到首页
