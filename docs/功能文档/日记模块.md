# 日记模块

> 映记的核心输入层：既支持 5 秒轻记录，也支持完整富文本日记。

---

## 模块定位

日记模块是映记最基础也最重要的功能，所有 AI 分析、成长中心、情绪星图、AI 助手记忆和社区内容都建立在日记数据之上。

当前日记模块已经拆成两个层级：

1. **今日心灯**：降低记录门槛，允许用户 5 秒完成一次状态采样。
2. **完整日记编辑**：当用户愿意深入表达时，再进入富文本编辑器。

这样避免用户一进入页面就面对完整编辑器，也避免把“连续记录”理解成“必须每天写长日记”。

---

## 路由

| 页面 | 路由 | 说明 |
|------|------|------|
| 日记列表 | `/diaries` | 查看所有日记，支持搜索和过滤 |
| 日记详情 | `/diaries/:id` | 查看单篇日记完整内容 |
| 今日心灯 / 新建日记 | `/diaries/new` | 默认展示今日心灯，可继续进入完整日记编辑 |
| 编辑日记 | `/diaries/:id/edit` | 修改已有日记，直接进入完整编辑器 |

---

## 今日心灯

今日心灯是映记新增的低门槛记录方式，目标是解决“用户不想写长日记”的问题。

用户只需要选择：

| 项目 | 示例 |
|------|------|
| 情绪 | 开心、平静、一般、低落、焦虑、烦躁、疲惫 |
| 能量 | 1~5 |
| 事件 | 学习、人际、家庭、身体、工作、其他 |

然后点击「完成今日心灯」，即可创建一条轻量日记。

生成内容示例：

```text
今天我点亮了一盏心灯。
情绪：焦虑；能量：2/5；事件：学习。
```

如果用户愿意继续表达，可以选择：

- 继续补一句
- 让 AI 问我
- 写完整日记
- 语音说说（入口保留）

### 今天不想写

用户也可以点击「今天不想写」。这不是简单跳过，而是会落库一条休息型轻记录：

```text
今天不想写，也是一种照顾自己。
```

产品含义：

- 主动意识到“今天不想表达”也是一种状态采样
- 不把用户逼向长文本
- 不让用户因为没有写正文而产生失败感

## 连续心灯与心灯护盾

### 为什么叫连续心灯

产品上不再使用“连续记录”作为主文案，因为它容易让用户理解成：

> 我必须每天写一篇完整日记。

现在改为 **连续心灯**，强调的是：

> 只要今天有一次有效照顾行为，就算点亮了今天的心灯。

有效行为包括：

- 5 秒情绪签到
- 一句话记录
- 完整日记
- 主动点击「今天不想写」

### 心灯护盾规则

心灯护盾是连续心灯的断签保护机制，不是心理分数。

规则：

| 规则 | 说明 |
|------|------|
| 初始护盾 | 每个用户默认 3 个 |
| 护盾上限 | 普通用户最多 3 个 |
| 漏记保护 | 用户某天完全没有有效行为时，可消耗 1 个护盾保护连续心灯 |
| 七天奖励 | 连续点亮心灯满 7 天，奖励护盾 +1 |
| 满盾处理 | 如果已经有 3 个护盾，则不再增加 |

后台会记录：

- 当前护盾数
- 已奖励过的连续心灯里程碑
- 已被护盾保护的日期

对应数据表：

```text
care_statuses
```

对应接口：

```http
GET  /api/v1/diaries/care/progress
POST /api/v1/diaries/care/rest
```

其中 `care/progress` 返回：

| 字段 | 说明 |
|------|------|
| `active_days` | 真实有记录的天数 |
| `actual_streak` | 不含护盾的真实连续心灯 |
| `protected_streak` | 含护盾保护后的连续心灯 |
| `shield_balance` | 当前护盾数量 |
| `shielded_days` | 被护盾保护的天数 |
| `shield_awarded` | 本次是否触发七天奖励 |
| `weekly_active_count` | 本周轻记录次数 |
| `weekly_goal` | 本周目标次数，当前为 3 |

## 日记包含哪些信息

| 字段 | 说明 |
|------|------|
| 标题 | 可选，不填时可由系统生成今日心灯标题 |
| 内容 | 正文，支持轻记录文本和完整富文本 |
| HTML 内容 | `content_html`，用于前端富文本渲染 |
| 日记日期 | 默认今天，可以改成其他日期 |
| 情绪标签 | 多选，如「开心」「焦虑」「平静」等 |
| 重要性评分 | 1~10 分，默认 5 |
| 图片 | 支持上传多张图片 |
| 是否分析 | 标记该日记是否已有 AI 分析 |

---

## 日记列表

- 分页展示，每页 20 条
- 支持按日期范围、情绪标签和关键词过滤
- 每条日记显示标题、日期、内容摘要、情绪标签、重要性评分
- 情绪标签会映射成中文显示，避免英文标签直接暴露给中文用户

## 完整日记编辑

完整编辑器基于 **Lexical** 构建，提供类似 Notion 的富文本编辑体验。

在 `/diaries/new` 中，完整编辑器默认不展开；用户点击「写完整日记」后才进入完整编辑模式。

在 `/diaries/:id/edit` 中，直接进入完整编辑模式。

### 自动保存

编辑器支持本地自动保存草稿，并提供明确状态提示：

- 正在保存
- 已自动保存
- 保存失败，请重试

自动保存的目的是避免用户写了很久后因刷新、误关闭等原因丢失内容。

### 富文本编辑器能力

#### 基础格式

- 加粗、斜体、下划线、删除线、行内代码
- 支持 Markdown 快捷语法
- 支持标题 H1~H6、引用、代码块、有序列表、无序列表

#### Notion 风格浮动工具栏

选中文字后弹出浮动工具栏，支持：

- 字号调节
- 文字颜色
- 背景高亮色
- 加粗、斜体、下划线、删除线、行内代码

#### Slash 命令

在编辑区输入 `/` 弹出快捷菜单，目前支持：

- 文本
- 标题
- 项目符号列表
- 编号列表
- 引用
- 代码块
- 插入图片

### 图片功能

- 顶部按钮选择图片上传
- Slash 命令插入图片
- Ctrl+V 粘贴截图或复制的图片
- 上传到后端 `uploads/diary_images/` 目录
- 文件名自动生成 UUID，防止冲突
- 支持格式：jpg、png、gif、webp

### AI 起题与每日映题

- AI 起题：基于日记内容生成标题
- 每日映题：通过后端引导问题接口提供写作提示，降低空白页压力

## 数据存储

日记主体仍存储在 `diaries` 表：

| 字段 | 用途 |
|------|------|
| `content` | 纯文本/Markdown 内容 |
| `content_html` | 富文本 HTML 渲染内容 |
| `emotion_tags` | 情绪标签数组 |
| `diary_date` | 日记归属日期 |
| `images` | 图片路径数组 |

心灯连续与护盾状态存储在 `care_statuses` 表。

保存日记本身不阻塞 AI 分析；单篇分析和综合冰山分析由 AI 模块按需触发。

## 技术实现

| 组件 / 模块 | 说明 |
|------|------|
| `DiaryEditor.tsx` | 今日心灯 + 完整编辑器页面 |
| `RichTextEditor.tsx` | 主富文本编辑器，基于 `@lexical/react` |
| `ImageNode.tsx` | 自定义 Lexical 图片节点 |
| `FloatingToolbarPlugin` | Notion 风格浮动格式工具栏 |
| `SlashCommandPlugin` | `/` 快捷命令插件 |
| `PasteImagePlugin` | Ctrl+V 粘贴图片插件 |
| `TopToolbarPlugin` | 编辑区顶部简版工具栏 |
| `diaries.py` | 日记、心灯进度、图片上传等后端接口 |
| `CareStatus` | 心灯护盾与连续状态模型 |

---

## 删除日记

- 支持删除单篇日记
- 删除后相关时间轴事件会清除
- 日记删除会影响后续统计、情绪分布和连续心灯计算
