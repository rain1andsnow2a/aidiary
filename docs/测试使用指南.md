# 测试使用指南

## 快速开始

### 运行所有单元测试
```bash
cd backend
pytest tests/ -v
```

### 运行特定测试文件
```bash
# 测试安全功能
pytest tests/test_security.py -v

# 测试邮件服务
pytest tests/test_email_service.py -v

# 测试认证服务
pytest tests/test_auth_service.py -v
```

### 运行真实邮箱测试
```bash
# 仅测试邮件发送
python tests/run_real_test.py --email-only

# 完整测试流程（需要手动输入验证码）
python tests/run_real_test.py
```

## 测试命令参考

### 基础命令

```bash
# 运行所有测试
pytest

# 详细输出
pytest -v

# 显示打印输出
pytest -s

# 显示错误堆栈
pytest --tb=short

# 只运行失败的测试
pytest --lf

# 遇到第一个失败停止
pytest -x
```

### 高级命令

```bash
# 运行特定测试类
pytest tests/test_security.py::TestPasswordHashing -v

# 运行特定测试方法
pytest tests/test_security.py::TestPasswordHashing::test_password_hashing -v

# 并行运行（需要安装pytest-xdist）
pytest -n auto

# 生成覆盖率报告
pytest --cov=app --cov-report=html

# 生成HTML报告
pytest --html=report.html --self-contained-html
```

## 真实邮箱测试指南

### 测试前准备

1. **确认应用已启动**
```bash
python main.py
```

2. **确认邮箱配置**
```bash
# 检查.env文件中的QQ邮箱配置
QQ_EMAIL=2337590486@qq.com
QQ_EMAIL_AUTH_CODE=lkvxwqnfjwvpdief
```

### 测试流程

#### 1. 邮件发送测试
```bash
python tests/run_real_test.py --email-only
```

**预期结果**:
- 邮件成功发送到 2337590486@qq.com
- 检查邮箱收件箱（可能在垃圾邮件）

#### 2. 完整注册测试
```bash
python tests/run_real_test.py
```

**步骤**:
1. 脚本发送注册验证码
2. 检查邮箱获取6位验证码
3. 输入验证码
4. 完成注册
5. 自动清理测试数据

**重要**:
- 测试完成后会自动删除测试用户
- 所有测试数据都会被清理
- 不影响生产数据

## 编写新测试

### 单元测试模板

```python
import pytest

class TestFeature:
    """测试功能"""

    def test_basic_behavior(self):
        """测试基本行为"""
        # Arrange
        input_data = "test"

        # Act
        result = process(input_data)

        # Assert
        assert result is not None
        assert result == expected

    @pytest.mark.asyncio
    async def test_async_behavior(self):
        """测试异步行为"""
        result = await async_function()
        assert result is True
```

### 集成测试模板

```python
import pytest
from httpx import AsyncClient, ASGITransport
from main import app

@pytest.mark.asyncio
class TestAPIEndpoint:
    """测试API端点"""

    async def test_endpoint_success(self):
        """测试成功响应"""
        transport = ASGITransport(app=app)
        async with AsyncClient(transport=transport, base_url="http://test") as client:
            response = await client.get("/api/endpoint")

        assert response.status_code == 200
        data = response.json()
        assert data["success"] is True
```

### 使用Fixtures

```python
import pytest

def test_with_database(db_session):
    """使用数据库会话"""
    # 创建测试数据
    user = User(email="test@example.com")
    db_session.add(user)
    await db_session.commit()

    # 测试
    result = await get_user(db_session, user.id)
    assert result is not None

def test_with_auth(auth_headers):
    """使用认证头"""
    response = await client.get("/api/protected", headers=auth_headers)
    assert response.status_code == 200
```

## 常见问题

### Q: 测试数据库在哪里？

A: 使用内存SQLite数据库，每个测试独立，自动清理

### Q: 如何调试异步测试？

A: 使用 `pytest -v -s` 查看输出，或使用 `pytest --pdb`

### Q: 真实测试会发送垃圾邮件吗？

A: 是的，使用你的真实邮箱。建议使用专门的测试邮箱

### Q: 如何跳过真实邮箱测试？

A: CI/CD中应该mock邮件服务，不执行真实测试

### Q: 测试失败怎么办？

A:
1. 查看错误堆栈: `pytest --tb=long`
2. 只运行失败测试: `pytest --lf`
3. 进入调试模式: `pytest --pdb`

## 测试最佳实践

### DO（推荐）

✅ 使用描述性的测试名称
✅ 一个测试只测试一个行为
✅ 使用Arrange-Act-Assert模式
✅ 测试边界条件
✅ 测试错误路径
✅ 保持测试独立
✅ 清理测试数据

### DON'T（不推荐）

❌ 测试实现细节
❌ 测试多个行为
❌ 依赖测试执行顺序
❌ 硬编码测试数据
❌ 忽略错误处理
❌ 跳过数据清理
